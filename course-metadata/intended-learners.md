## What will students learn (learning objectives)

- Design agentic loops driven by stop_reason and avoid anti-patterns that break control flow in production Claude systems (Domain 1) (130)
- Implement hub-and-spoke multi-agent systems with isolated subagent context, parallel Task spawning, and structured handoffs (Domain 1) (134)
- Apply programmatic enforcement vs prompt guidance correctly for financial, compliance, and identity-verification workflows (Domain 1) (133)
- Write MCP tool descriptions and structured error responses that drive reliable tool selection and recovery decisions (Domain 2) (127)
- Configure CLAUDE.md hierarchy, .claude/rules/ globs, slash commands, and skills with context: fork for team workflows (Domain 3) (128)
- Engineer prompts with explicit criteria, few-shot examples, and tool_use JSON schemas for reliable structured output (Domain 4) (127)
- Debug context degradation using case-facts blocks, scratchpads, and claim-source provenance in long-running agents (Domain 5) (125)
- Pass the Anthropic Claude Certified Architect exam via a 40-question practice exam mapped to all 5 domain weights (Section 9) (125)

## Prerequisites

- Basic familiarity with LLM APIs (OpenAI, Anthropic, or equivalent)
- Python 3.10+ OR Node.js 18+
- JSON and JSON Schema syntax
- Command line fluency
- Claude Code CLI installed via `npm install -g @anthropic-ai/claude-code`
- An Anthropic API key (optional, for hands-on labs)

## Who is this course for

- Solution architects preparing for the Anthropic Claude Certified Architect exam who need domain-by-domain coverage mapped to the official exam guide (148)
- Senior engineers building production AI agents with Claude, MCP, and the Agent SDK who want depth on tool design, orchestration, and reliability (144)
- Technical leads evaluating Claude Code for team adoption — CLAUDE.md hierarchy, .claude/rules, skills, slash commands, and CI/CD integration patterns (149)
