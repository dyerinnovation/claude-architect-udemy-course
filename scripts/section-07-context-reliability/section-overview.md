# Section 7: Domain 5 — Context Management & Reliability (15%)

## Overview
- **Domain**: Domain 5: Context Management & Reliability
- **Exam Weight**: 15%
- **Lecture Count**: 17 lectures
- **Estimated Duration**: ~2 hours

## Learning Objectives

By the end of this section, students will be able to:
- Counter the "lost in the middle" effect through trimmed tool outputs, persistent "case facts" blocks, and disciplined progressive summarization.
- Apply an escalation decision framework that honors explicit customer requests for human agents and treats sentiment and complexity as independent signals.
- Implement structured error propagation across multi-agent systems and distinguish access failures from valid empty results.
- Design crash recovery and cross-context persistence using scratchpad files, structured agent state manifests, and coverage annotations in synthesis output.
- Evaluate accuracy and provenance using stratified sampling, claim-source mappings, and explicit handling of conflicting sources rather than arbitrary choice.
